Preentrega 2
Para poner en práctica todos los conocimientos adquiridos, te proponemos avanzar con la estructura avanzada y el uso colaborativo con GitHub.

Objetivos
Realizar una estructura avanzada de la web.


Efectuar el estilo avanzado de la web.


Generar la carga en un repositorio en GitHub.


Requisitos
Este trabajo cuenta con tres instancias (puedes compartir la URL de tu repositorio de GitHub con las tres):

Estructura del HTML avanzada

Realizar una estructura del HTML prolija, limpia y fácil de leer.


Agregar elementos HTML según la necesidad de armar contenedores o elementos web determinados, basándose en el framework elegido y la documentación del mismo.


Usar Bootstrap o Grids + Flexbox.


Maquetar las estructuras en la web basándose en el framework elegido, haciendo uso de clases utilitarias para armar grillas, elementos web y estilos propios del framework, además del HTML de contenido.


Todas las páginas deben tener el contenido estructurado y el estilo linkeado.


Agregar las diferentes librerías de JavaScript y CSS pertinentes al framework.


La estructura debe ser entregada en uno o varios archivos HTML según tu proyecto.



Archivo de CSS con propiedades avanzadas

Tu archivo CSS debe contar con:

Estilo del framework: ajustar con CSS la estética de los componentes para que condigan con el sitio.


Estructura de la web: uso de etiquetas no solo para el contenido, sino para conformar el layout, contenedores, etc.



Las propiedades avanzadas deben ser entregadas en un archivo CSS.



Git y GitHub para brindar acceso al proyecto versionado

Tu repositorio debe contener:

Archivos: todos los necesarios para visualizar correctamente la web.


Historial de versionado: mostrar los commits usados para actualizar/versionar el código.



Proporcionar el acceso público al repositorio mediante la URL del mismo.

Diseñando nuestra página web desde cero: prototipado, maquetado y estilización inicial
Recomendaciones

Optimiza tu sitio para velocidad y eficiencia: compresión de archivos, uso de CDN y optimización de imágenes.


En GitHub, sigue buenas prácticas de control de versiones: ramas separadas para features, fixes y commits descriptivos.


Pruebas cross-browser y dispositivos: verifica en navegadores y tamaños de pantalla diversos. Herramientas útiles: BrowserStack o CrossBrowserTesting.


Criterios de evaluación
HTML
Prolijidad:
Tabulaciones correctas y ordenadas.

Jerarquía clara entre elementos.

Comentarios usados de forma pragmática para delimitar secciones o realizar comentarios asertivos.


Tags:
Anidado óptimo.

Uso correcto de la semántica y estructuración.

La propiedad alt en las imágenes es obligatoria.


Estilo y funcionalidad:
Nombres de clases correctos, evitando redundancias.

Enlaces operativos.

Imágenes pertinentes al contenido. Bien cargadas.

Rutas relativas correctas.


Contenido:
Contenido correctamente estructurado (tablas, listas, títulos, párrafos, imágenes).

Presentar niveles de lectura para evitar la monotonía.

Las imágenes deben tener un tamaño acorde al espacio que ocupan.


<--------------->

CSS
Código y diseño de la estructura visual:
Distribución visual atractiva, evitando monotonía.

Jerarquía tipográfica y de espacios.

El tamaño de las imágenes debe ser proporcional a su contenedor.


Diseño web:
Paleta de colores con contrastes pertinentes y consistentes.

Texto legible.


Prolijidad del código:
Tabulaciones correctas.

Uso eficaz de selectores para evitar la repetición de código.


Comprensión del código:
Uso de clases modificadoras.

Aplicar el diseño responsive de forma correcta.


GitHub
Git:
Commits coherentes agrupando mejoras.

Creación de ramas para pruebas.

Merge a main.

Uso de “.gitignore” para excluir archivos/carpetas no requeridos.


GitHub:

Uso de GitHub Pages para publicar el trabajo.
